二者共同回答「Agent 正在怎样完成我的任务」——不是减少信息，而是把信息放进正确的注意层级

为什么不能只保留对话

  • 对话是时间序列，无法稳定表达并行步骤与依赖
  • 工具日志持续写入主流，会淹没真正需要理解的决策
  • 只展示最终回复，长任务又会变成黑盒

任务概览必须提供

  • 任务进展、执行项与待办
  • 任务产物与验收入口
  • Skills / MCP 等资源状态
  • 上下文与 Token 消耗

设计原则

默认保护任务主线；只有会改变方向、风险、成本或交付结果的信息，才进入主要注意层级。

对页面的含义

DESIGN PATTERN
Assistant 时代，我们设计 AI 如何进入人的操作。
Agent 时代，我们要设计人如何把工作交出去，又不失去控制。

新的交互范式是「可监督的任务委托」：以任务而非消息为核心，以阶段和异常而非全部日志建立过程透明，以真实产物和证据而非完成声明建立信任——三域由同一任务状态持续联动。

TASK左 · 跨任务管理 COORDINATION中 · 协作与控制 ARTIFACT右 · 产物与验收

Agent 工作台的价值，不是让 AI 更像一个会聊天的人，
而是让人能够像管理真实工作一样，管理 AI 的持续行动与交付。
