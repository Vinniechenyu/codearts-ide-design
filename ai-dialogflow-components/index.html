Agent 对话式交互指南

Agent 对话流的核心矛盾：过程要透明，但不能淹没结果。用户既要知道 Agent 正在做什么、能在出错和高风险时介入，又不想被一屏滚动的工具日志埋掉最终答案。这套库用「过程默认收拢、结果优先、写操作可控、状态强一致」四条主张，把 chat / code 全过程拆成 消息容器 · 内容块 · 过程层 · 控制汇总 四层，沉淀为一套可复用、可落地的对话交互组件。

设计原则

Foundation

作为 Foundation 层，解释组件状态与交互规则背后的判断依据。具体怎么渲染看各组件，为什么这么渲染看这里。

P01过程可见，但默认不打扰

Agent 执行过程需要让用户知道正在发生什么，但完成后应自动收拢，避免污染最终答案。

Do

进行中展示关键步骤与当前动作；完成后折叠为耗时摘要；失败时自动展开错误原因。

Don't

把所有工具调用长期平铺在主对话里；用过多动画表达忙碌；让用户必须读过程才能理解结果。

关联组件3.13.24.14.10
P02结果优先，过程次之

对话流的主线应该服务于最终答案和可用产物，过程信息只在诊断、审查或学习时展开。

Do

最终结论、产物链接、已实现内容放在主阅读路径；过程细节默认弱化或折叠。

Don't

让执行日志抢占答案位置；把内部推理、工具参数、长日志直接暴露为主要内容。

关联组件2.13.24.14.94.10
P03写操作必须可控

凡是会修改文件、执行命令、联网发送、删除或改权限的行为，都要让用户有控制感。

Do

高风险操作先授权；文件变更可查看 diff；失败可重试；完成后产物可追踪。

Don't

静默执行高风险命令；只告诉用户「已完成」但不给变更明细；授权按钮不区分风险等级。

关联组件3.33.44.54.74.9
P04状态表达必须一致

同一种状态只使用一套颜色、图标、动词和行为规则，避免用户重新学习。

Do

进行时用 info 蓝 + loader；完成用 success 绿或收起灰；失败用 danger 红；待确认用 warning 琥珀。

Don't

同一状态混用不同颜色；「排队中」和「进行中」都用旋转 loading；失败态没有重试入口。

关联组件全局状态机3.1–3.74.14.64.10
P05降低用户决策成本

当 Agent 需要用户补充信息时，应优先给可选项和推荐项，而不是抛开放问题。

Do

给 2–4 个互斥选项；标出推荐；允许「其他」自由输入；显示当前步骤数。

Don't

一次问多个开放问题；没有默认推荐；让用户不知道选完之后会发生什么。

关联组件1.24.24.44.11
P06模式边界清晰

Chat 模式偏解释、总结、内容展示；Code 模式偏执行、文件、命令、diff、授权，两者组件可复用但语义不能混淆。

Do

只读代码块和生成型代码块视觉相近但标签明确；展示型终端和真实运行终端明确区分。

Don't

把只读示例做成像真实执行；把真实写入操作包装成普通文本输出。

关联组件2.32.43.33.44.54.7
P07可恢复优于一次成功假设

Agent 工作流必须默认考虑失败、中断、取消和排队，而不是只设计成功路径。

Do

排队可取消；进行中可停止；失败可重试；中断可继续；完成后可重新执行。

Don't

只有成功态；取消后没有下一步；连接中断和任务失败使用同一种表达。

关联组件3.54.64.10

全局交互状态机

L3 / L4 共用

状态机定义所有过程型组件的颜色、图标、可展开性和主操作，是 L3 / L4 组件的共用行为规则。用户在任何过程块前的判断逻辑保持一致：进行时必有「停止」，失败必有「重试」，省略号只出现在进行时。

进行时 In progress --info ti-loader-2 仅当前步 停止 / 中断 工具 / 生成执行中

色彩准则 语义色只用四种——info 蓝（进行）、success 绿（完成）、danger 红（失败）、warning 琥珀（待确认）；排队 / 取消归为中性灰，靠删除线、占位区分，避免「三种灰」互相混淆。「排队中」另抽成组件 3.5 排队中卡片（见 L3）。

消息数据模型

Spec

组件背后的消息结构：一条消息由 role / status / contentType / actions 四个维度描述，组件只是这些字段的可视化呈现。

字段取值

statusqueued running completed failed cancelled needsApproval
contentTypetext code terminal artifact citation diff media
actionsretry stop approve viewDiff copy continue

动效规范

Foundation

动效只服务于状态表达——让用户一眼分清进行 / 完成 / 可展开，而非炫技。时长集中在两档：微交互 .12–.3s、持续态循环 1–1.8s；缓动统一 ease / linear。全部尊重 prefers-reduced-motion：关闭循环与位移。
