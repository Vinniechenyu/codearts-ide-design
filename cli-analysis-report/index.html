01 · 原生集成
无需切换上下文
零上下文切换终端原生
CLI 直接在开发者已有的终端中运行。无需浏览器、无需 GUI 窗口——AI 能力就在代码所在的地方，思维链路不中断。
02 · 可编排性
与工程工作流无缝融合
Shell 原生CI/CD 集成
天然支持管道、重定向和脚本化，可嵌入 Makefile、GitHub Actions、pre-commit hooks，真正成为工程流程的一部分。
03 · 效率优先
键盘驱动，毫秒启动
低摩擦极速响应
没有 GUI 渲染开销，没有 Electron 启动延迟。追求极致效率的开发者需要的是最短路径——从想法到执行，没有多余摩擦。
背景洞察 02 · 现状审查 OpenCode 的设计局限 — 设计优化的必要性
01 · 启动阶段
Banner 信息结构薄弱
启动时仅输出基础版本信息，缺乏工具状态、权限模式、模型上下文等关键启动态。开发者无法在启动瞬间建立完整的会话认知。
无权限模式指示
无模型 / Token 预算展示
02 · 表达意图
上下文管理能力缺失
输入框为纯文本，缺少 @file / @url 等结构化上下文注入机制，也没有 Context Chips 让开发者直观管理已添加的上下文范围。
无 @mention 上下文注入
无 Context Chips 可视化
03 · 等待执行
执行透明度不足
Tool Call 以纯文本流输出，缺乏结构化折叠展示；无清晰进度指示器；上下文压缩等关键事件静默发生，开发者无感知。
Tool Call 无结构化展示
上下文压缩无提示
04 · 审查确认
Diff 与审查体验简陋
代码变更以原始 diff 输出，无语法高亮；Accept / Reject 操作缺乏清晰的交互流；开发者需离开 CLI 去 IDE 中确认变更。
Diff 无语法着色
无 Accept / Reject 工作流
05 · 视觉 / 主题
视觉层次与品牌感弱
全局单色终端输出，错误、警告、成功等信息与普通文本视觉权重相同；缺乏主题系统，无品牌识别度。
无语义化颜色体系
缺乏视觉层次设计
06 · 多智能体
多 Agent 协作支持有限
多个 Agent 并行运行时无统一管理面板；子 Agent 状态、任务分配、进度跟踪均不可见，协作效率难以保障。
无 Agent 管理面板
子任务状态不透明

Page Framework · 页面框架 CLI 页面框架 — 我们的方向
推荐页面框架：固定分离式 + 轻量 Block
以 Claude Code 的上下分离为基础，叠加 Warp 的 Block 可操作性概念，在保持兼容性的前提下最大化信息密度与可操作性。
1
固定底部输入区
输入框始终定位在底部 20% 区域，包含 Context Chips、模型切换与费用角标，输入位置永远稳定，降低用户视线跳跃成本。
2
轻量 Block 标注
仅对工具调用产生的写入操作输出加可折叠 badge，不依赖 GPU 渲染，兼容 SSH/tmux，同时提供类 Warp 的结构化感知。
3
顶部轻量状态栏（可选）
参考 Warp 的 status bar 概念，在顶部用单行显示当前模型、Token 用量与 Git branch，不占用主内容区。
4
渐进式升级路径
先落地固定分离式框架（与现有 Claude Code 完全一致），再逐步引入 Block badge，避免一次性重构风险。
总结：固定分离式框架成本最低、兼容性最强，是最合理的起点。轻量 Block 标注作为第二阶段叠加，可显著提升工具调用可读性，同时不破坏现有 PTY/SSH 场景。

Multi-Agent · Our Direction多智能体架构 — 我们的方向
Hub & Spoke 起步 + P2P 演进 + 子 Agent 上下文隔离
以 Hub & Spoke（Subagents）起步，实现成本最低；随产品成熟演进到 P2P Mailbox（Agent Teams 架构）。子 Agent 在独立上下文中执行，只向主 Agent 返回摘要，控制 token 成本。
01
Hub & Spoke 起步：主 Agent 委派子任务
主 Agent 负责规划和决策，子 Agent 负责执行（读文件、搜索、代码分析等探索性任务）。子 Agent 自动使用低成本模型（Haiku），完成后向主 Agent 返回摘要而非原始输出，避免主上下文被污染。
02
等待状态：暂停标记，不猜测继续
Agent 遇到权限不够或需要澄清时，自动暂停并标记橙色" 等待输入"，向用户发送通知。不会猜测"用户可能想要什么"而自动继续——这是"安全自主"的核心原则，避免雪球效应。
03
统一任务面板，人随时可介入任意 Agent
所有 Agent 任务在统一面板中可见，人可以点击任意任务查看详情或给新指令。不是"启动了就不管"，而是"随时可介入"。人是所有 Agent 的最终决策者，不是被排除在外的观察者。

Touchpoint · Our DirectionAgent 管理面板 — 我们的方向
统一任务面板 + 等待状态可视化 + 上下文隔离
多 Agent 有统一的任务管理面板，每个 Agent 有状态标记（运行中/等待/完成/失败）。Agent 遇到不确定时暂停等待而非猜测继续。子 Agent 在独立上下文中执行，主 Agent 只收摘要。
01
统一任务面板，状态一眼扫全局
所有 Agent 任务列在一个面板里，颜色和图标区分状态：运行中（绿色动画）、等待确认（橙色警告）、已完成（灰色）、失败（红色）。步骤进度条 + 耗时显示。可点击任意任务进入详情。

优势 无额外信息，退出最快速 每条费用透明，随时可查 强化完成感，清楚知道这次做了什么费用意识：知道这次用了多少钱 输出简洁
劣势 无收尾感，会话虎头蛇尾费用完全不可知 分散的单次摘要需自己汇总无完整的会话级总结 无任何收尾感
总结缺乏收尾感，不推荐透明但缺乏整体感我们的推荐方案无收尾，最差
推荐：退出时展示会话摘要卡（完成任务数 / 修改文件数 / 总 token / 总费用 / 用时），强化每次会话的完成感。摘要卡可展开查看按任务归因的费用明细，帮助用户了解哪类任务最值得用 AI。
Touchpoint · Our Direction会话摘要卡 — 我们的方向
退出时自动展示 + 按任务归因 + 可分享
用户退出会话时自动展示摘要卡：完成任务数、修改文件数、总 token 用量、总费用、总时长。摘要卡可展开查看按任务归因的明细，可选择复制摘要或分享（适合团队汇报）。
01
退出时自动弹出，不需要主动查
用户输入 exit 或关闭终端时，摘要卡自动弹出显示 3-5 秒（或用户主动关闭）。不需要用户记得去查，做完了自然就知道这次花了多少、做了什么。强化每次会话的完成仪式感。
02
展开查看按任务归因的明细
摘要卡默认显示汇总数据，点击展开后显示每条任务的费用归因（任务描述 + token + 费用）。让用户了解"哪类任务最贵"，下次可以有意识地用更便宜的模型处理简单任务，主动控制成本。
03
可复制摘要，适合团队汇报
摘要卡右上角有"复制摘要"按钮，生成适合粘贴到 Slack/PR 描述的文本（"用 AI 完成了 3 个任务，修改了 5 个文件，用时 24 分钟"）。降低汇报 AI 工作成果的摩擦。

Design Summary · 设计总结 从用户诉求到设计点 — 完整链路
User Needs
用户诉求
快速开始
无需繁琐配置，30 秒内启动第一个任务，不被工具本身拖慢节奏
执行透明
清楚知道 AI 在做什么、做到哪一步，不产生"黑盒焦虑"
随时介入
出现偏差时能快速中断、修正方向，而不是等 AI 跑完再收拾残局
费用可感知
每次操作的 token 消耗和费用实时可见，能主动控制成本
Pain Points
用户痛点
启动信息冗余
Banner 噪音多于信号，版本号、Tips、广告挤占注意力，认知负担高
执行黑盒
等待时不知进度、不知做到哪步，长任务中焦虑感持续累积
高危无预警
删文件、执行脚本等高风险操作缺乏分级提示，事后难撤回
费用不透明
单次和累计消耗均不可见，无法评估 AI 的使用价值与 ROI
多 Agent 失控
并行任务状态分散，无统一视图，Agent 卡住时无感知也无入口处理
UX Goals
体验目标
透明
AI 的每一步行为都有迹可循，用户永远不处于信息真空中
掌控
用户永远是最终决策者，任何时刻都能中断、修正、介入任意任务
效率
减少认知摩擦与等待焦虑，让注意力聚焦在任务目标而非工具操作上
信任
行为可预期、结果可解释、操作可撤销，长期建立用户与 AI 的信任关系
Opportunities
体验机会点
极简 Banner
仅显示模型名 + 权限模式两个关键信号，消除其余噪音，降低启动门槛
结构化执行流
读取类 Tool Call 自动折叠计数，写入类展开加 Block badge，让过程可读可扫
风险分级弹窗
权限确认颜色编码（蓝/橙/红），高危操作触发 Plan Mode 先提案再执行
统一任务面板
多 Agent 状态一屏掌控，等待中的 Agent 橙色暂停标记，非阻塞处理
会话摘要卡
退出时展示完成任务数/费用/用时，强化完成感，驱动费用意识
Design Actions
设计点
固定底部输入区
输入框定位底部 20%，含 Context Chips、模型切换、费用角标，位置永远稳定
Thinking 折叠 + Block 展开
Thinking 默认一行摘要可展开；写入操作 Block 展开标色，完成后可折叠
弹窗风险配色
蓝色=读取，橙色=写入，红色=执行；三档授权 + "修改后执行"第四选项
非阻塞暂停标记
Agent 等待时橙色标记 + 桌面通知，不卡死其他任务，用户主动处理
退出摘要 + 费用归因
退出时摘要卡自动弹出，可展开按任务费用明细，可复制分享给团队
总体原则
让「AI 在做什么」始终可见，让「用户能做什么」始终触手可及。透明度不是噪音，而是信任的基础。自动化程度不是目标，掌控感才是。每一个设计决策都应该问：「这让用户更有掌控感了吗？」

PHASE 03 · 等待执行（情绪最低谷）
Thinking 默认一行摘要，点击展开，完成后自动折叠

高危操作自动触发 Plan Mode 先提案再执行
步骤计数（n/N）+ 整体进度条，双重反馈
失败高亮红色 + "让 AI 修复"快捷入口内嵌进度视图
Esc 中断按钮始终浮动可见，不需记快捷键

弹窗颜色编码风险等级（蓝色读取 / 橙色写入 / 红色执行）

默认输出摘要行，变更 ≥ 3 文件自动建议 Review Panel
输出末尾恒有追问入口（✦），驱动自然迭代节奏
费用角标始终在 Chip 区可见，点击展开按任务明细
